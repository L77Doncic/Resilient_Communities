# 智慧应急网站构建计划（Resilient Community Web）

## 1. 项目背景与目标

- 主题：智慧应急——气候变化背景下的韧性社区
- 展出时间：2026年5月12日-5月18日（第18个全国防灾减灾周）
- 展出地点：四川大学灾后重建与管理学院展厅
- 展出形式：网站（现场二维码扫码访问）

### 1.1 核心目标

1. 建设一个可展可评的完整网站系统，覆盖“科普展示 + 互动参与 + 评分点评”。
2. 在视觉上统一且有记忆点，严格遵循 `DESIGN.md` 的风格规范。
3. 通过 AI 智能体提升内容生产、交互创意、测试效率与讲解质量。
4. 在展期高并发扫码场景下稳定运行，支持老师与同学快速打分和评论。

### 1.2 目标用户

- 现场观众：老师、同学、学院访客
- 评审群体：需要扫码打分并点评
- 传播用户：通过链接二次访问的非现场用户

## 2. 产品范围（MVP + 展示增强）

### 2.1 MVP（必须完成）

1. 首页（主题叙事 + 入口导航）
2. 气候风险认知模块（图文/数据卡片）
3. 韧性社区解决方案模块（分场景策略）
4. 互动体验模块（风险自测/应急任务闯关二选一）
5. 作品评分与点评模块（扫码后可提交）
6. 关于与致谢页面（团队、方法、AI 使用说明）
7. 后台轻量管理（评分与评论数据导出）

### 2.2 展示增强（优先级高）

1. 数据可视化故事线（时间轴、对比图、热力图）
2. AI 讲解助手（FAQ 引导 + 模块推荐）
3. 个人“韧性画像”生成卡（可分享）
4. 展厅大屏模式（自动轮播与讲解脚本）

## 3. 功能结构

1. `首页 Home`
- 主题宣言、展会信息、快速入口
- “立即体验/立即评分”双 CTA

2. `认知 Learning`
- 气候变化与灾害风险基础知识
- 本地化社区脆弱性与韧性要素

3. `方案 Solutions`
- 家庭、校园、社区、城市四层解决方案
- 每层提供“低成本可执行清单”

4. `互动 Lab`
- 韧性自测（题目 + 即时反馈）
- 应急任务模拟（情景决策）

5. `评分与点评 Review`
- 维度评分（创新性、实用性、美观性、传播性）
- 文本点评与身份标签（老师/同学/访客）

6. `成果与下载 Results`
- 研究海报、报告摘要、素材下载

## 4. 技术方案

### 4.1 前端

- 框架：Next.js 15（App Router）
- 语言：TypeScript
- 样式：Tailwind CSS + CSS Variables（承接 `DESIGN.md`）
- 动画：Framer Motion（实现标志性 rotate + hard-shadow hover）
- 图表：ECharts / Recharts（二选一）

### 4.2 后端与数据

- BFF/API：Next.js Route Handlers
- 数据库：PostgreSQL（展前可降级 SQLite）
- ORM：Prisma
- 鉴权：仅管理端登录（观众匿名评分 + 防刷限制）
- 缓存：Redis（可选，用于限流与热点数据）

### 4.3 部署

- 首选：Vercel（快速迭代 + CDN）
- 备选：学校服务器 Docker 部署
- 域名：短链 + 二维码（展厅海报、桌牌）

## 5. 设计与体验要求

1. 全站必须遵循 `DESIGN.md`：暖底色、命名色板、字体层级、圆角、边框、阴影体系。
2. 保持“科普可读性 + 展示记忆点”的平衡：每屏仅一个主信息。
3. 所有核心交互 2 步内完成（进入、操作、反馈）。
4. 移动端优先：扫码后首屏加载 < 2.5s（4G 环境目标）。
5. 评分表单必须“短平快”：30 秒内可提交。

## 6. AI 智能体应用策略（原创与合规）

1. 文案智能体：生成科普文案初稿，人工二次校对并标注来源。
2. 视觉智能体：生成插画草图/图标风格探索，最终统一重绘与改编。
3. 数据智能体：辅助生成图表描述与解释文案。
4. 测试智能体：自动生成边界用例（表单、防刷、移动端适配）。

### 6.1 抄袭防控

1. 所有 AI 生成内容必须进行“二次改写 + 本地语境化”。
2. 对外部图文保持来源记录，形成 `credits` 清单。
3. 关键视觉（首页主视觉、核心流程图）必须原创重构。

## 8. 验收标准

### 8.1 功能验收

1. 核心 6 大模块可访问、可交互、可回流数据。
2. 评分与点评可提交、可查看统计、可导出。
3. 移动端适配通过主流机型（iOS/Android）验证。

### 8.2 体验验收

1. 视觉一致性：组件样式与 `DESIGN.md` 偏差 <= 10%。
2. 可读性：核心文案可在 3 分钟内完成快速浏览。
3. 互动性：至少 1 个“可分享”结果页面。

### 8.3 稳定性验收

1. 首屏 LCP < 2.5s（展厅网络条件下）。
2. 表单提交成功率 > 99%。
3. 异常降级：接口失败时提供本地兜底提示。

## 9. 风险与预案

1. 网络波动：启用静态缓存 + 本地离线应急页。
2. 并发提交高峰：限流与队列化写入。
3. 内容争议：管理端可一键隐藏不当评论。
4. 时间不足：先保 MVP 闭环，再做展示增强。

## 10. 交付清单

1. 线上网站（含二维码访问）
2. 代码仓库（前后端一体）
3. 设计规范与组件文档
4. 讲解脚本（3分钟版 + 8分钟版）
5. 展期运维手册（值守、备份、故障恢复）
